## Investor Room v6 — the capture surface becomes a conversion system

v5 turned the Investor Room into a cinematic, intent-aware capture surface. v6 turns it into a full investor pipeline: every lead is scored, every lead triggers two transactional emails, and every lead lands in a hidden admin dashboard. Allocation urgency, direct calendar booking, and a billionaire-grade founder video frame round out the ask.

**Stack position:** `feat/investor-room-v6` → `feat/investor-room-v5` → `feat/investor-room-v4` → `feat/investor-room-v3` → `feat/investor-room-v2` → `feat/investor-room-v1` → `main`.

### Priority deck (what shipped)

**1 · Billionaire-grade founder video frame**
- Cinematic 1px letterbox ring on black canvas
- Corner ticks (cinema framing marks)
- Oversized drop shadow + radial vignette behind the section
- Timecode strip below the frame (`TC · 00:00 → 01:30 · one take`)
- Placeholder upgraded: aqua glow pool, horizontal scan line, heartbeat dot with glow, "filed privately" caption
- When `NEXT_PUBLIC_FOUNDER_CALENDAR_URL` is set but `NEXT_PUBLIC_FOUNDER_VIDEO_URL` is not, the left rail pivots from "watch" to "book" — a primary book-a-call CTA carries the surface until the video lands.

**2 · Real HubSpot + Airtable env integrations**
- HubSpot custom properties written on every capture: `aissisted_intent`, `aissisted_score`, `aissisted_score_label`, `aissisted_score_points`, `aissisted_check_size`, `aissisted_note`.
- Airtable columns written: `Intent`, `Score`, `Score Points`, plus the existing contact columns.
- Env-gated. If keys are unset, the channel is logged as skipped with a structured reason — never throws, never blocks the 202.

**3 · Investor scoring (hot / warm / cold)**
- `lib/investor-scoring.ts` — deterministic, explainable `scoreLead()` with signal weights:
  - Check size (strongest): `$1M+` → +5, `$500K–$1M` → +4, `$250K–$500K` → +3, `$100K–$250K` → +2, `<$100K` → +1
  - Intent: `allocation` → +3, `founder-call`/`founder-session` → +2, `thesis-memo` → +1, `waitlist` → 0
  - Firm present → +2
  - Note length: ≥40 chars → +1, ≥200 chars → +2 (stacks)
- Thresholds (intentionally earned):
  - **Hot** ≥ 8 pts
  - **Warm** ≥ 4 pts
  - **Cold** < 4 pts
- Reasons are attached to every score so the subject line, admin chip, and HubSpot log are all explainable.

**4 · Transactional email fan-out (Resend)**
- `lib/email.ts` — plain-text only (investor inboxes strip HTML heavily). British-elite brand voice, no exclamation marks.
- `sendInvestorConfirmation(lead)` — intent-specific copy across six variants, signed from Ron.
- `sendFounderNotification(lead)` — subject carries the bucket: `[HOT]` / `[WARM]` / `[cold] Investor lead · name · firm · intent`. Body contains the full lead context and score reasons. `Reply-To` is set to the investor's email, so a one-click reply lands in their inbox.
- Env-gated: `RESEND_API_KEY` + `INVESTOR_FROM_EMAIL` + `FOUNDER_NOTIFY_EMAIL`. If unset, emails are logged as skipped and the endpoint still returns 202.

**5 · Hidden admin pipeline dashboard**
- Route: `/_/pipeline` — server component, token-gated via `ADMIN_PIPELINE_TOKEN`.
- Underscore-prefix + `metadata: { robots: { index: false, follow: false } }` keeps it out of sitemaps and bots.
- First visit with `?token=…` seeds an httpOnly + secure + lax cookie and redirects to the clean URL. Subsequent visits read the cookie.
- Renders: **Total / Hot / Warm / Cold** summary bar → latest 100 Airtable leads sorted by `Received At` desc → score chips, mailto links, formatted timestamps.
- If `ADMIN_PIPELINE_TOKEN` is unset, the entire surface 404s (invisible to unauthenticated visitors).
- Readonly. Reads Airtable only. Never writes. Never aggregates PHI.
- New API: `GET /api/investor/admin/leads` — token-validated Airtable proxy that powers the dashboard (Bearer / cookie / query token accepted).

**6 · Allocation urgency (honest scarcity)**
- New route: `GET /api/investor/allocation-status` — reads `INVESTOR_COHORT_SEATS_TOTAL` + `INVESTOR_COHORT_SEATS_FILLED` + optional `INVESTOR_COHORT_NEARLY_PCT` (default 70).
- Returns state: `open` / `nearly-full` / `closed` / `unconfigured`.
- New component: `AllocationUrgency` — surfaces a subtle seats-remaining chip directly above the `HardCTA` band.
  - Aqua dot when open
  - Brand-red pulse (`animate-ping`) when nearly full
  - Hairline when closed
- **If env is unset, renders nothing. No fabricated scarcity.**

**7 · Calendar booking integration**
- New component: `FounderCalendar` — reads `NEXT_PUBLIC_FOUNDER_CALENDAR_URL` + `NEXT_PUBLIC_FOUNDER_CALENDAR_LABEL` (default "Book on calendar").
- Surfaces as a secondary "book-direct" CTA inside the capture modal for `founder-call` / `founder-session` / `allocation` intents.
- Also surfaces as the primary CTA on the `FounderVideo` placeholder when the video is still in production — converting a "watch" beat into a "book" beat while the video is pending.
- Opens in a new tab (`rel="noopener noreferrer"`). Honest note: no cookies set on aissisted.com.
- **If env is unset, renders nothing.**

**8 · Copy tighten (world-class investor level)**
- `HardCTA` tiers:
  - "Request allocation." → **"Take your seat."**
  - "Book a founder call." → **"Thirty minutes. On camera."**
  - "Join the strategic waitlist." → **"Hold a seat for later."**
- `INTENT_COPY` modal copy rewritten across all six intents. Shorter verbs, tighter framing:
  - "The round is kept small on purpose."
  - "Working session — not a pitch."
  - "Every reply comes from the founder."
- Chapter 08 heading: "The next step is a conversation. Not a term sheet." → **"This isn't a pitch. It's an invitation."**
- Three-ways-in subhead: "Confirmation hits your inbox inside a minute. The real reply lands inside a business day."

### Brand discipline

- No fabricated metrics. `AllocationUrgency` is honest-or-silent. `FounderVideo` ships a designed placeholder, not a broken player.
- No emoji. No "AI-powered" framing.
- Aqua stays inside the 8% accent budget.
- All motion respects `prefers-reduced-motion`.

### Data isolation

- Investor console + every CTA endpoint use `surface: "investor"` with `noopMemoryAdapter`.
- Admin dashboard reads Airtable only. Never writes, never aggregates PHI.
- `/_/pipeline` is 404 by default — invisible unless `ADMIN_PIPELINE_TOKEN` is configured.

### Decision flags for @rongibori

1. **Resend + cohort env vars** — v6 is inert without `RESEND_API_KEY`, `INVESTOR_FROM_EMAIL`, and `FOUNDER_NOTIFY_EMAIL`. Scarcity chip is inert without `INVESTOR_COHORT_SEATS_TOTAL/FILLED`. Silent-by-default posture means no visual regressions if you ship the PR before configuring env.

3. **Calendar + video URLs** — paste `NEXT_PUBLIC_FOUNDER_CALENDAR_URL` when Cal.com is live; the left rail auto-promotes book-a-call until `NEXT_PUBLIC_FOUNDER_VIDEO_URL` is set.
