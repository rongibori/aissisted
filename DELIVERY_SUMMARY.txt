================================================================================
CRITICAL NOTES
================================================================================

1. DATE HANDLING IS CRITICAL
   - PostgreSQL timestamp columns expect Date objects, not ISO strings
   - Script does NOT auto-fix these (too many edge cases)
   - TypeScript type checking will catch missing fixes
   - See docs/EXECUTION_GUIDE.md Phase A Step 6 for full list

2. SCRIPT DOES NOT DEPLOY AWS
   - Script only creates infrastructure as code templates
   - AWS deployment requires separate manual process
   - See infra/aws/README.md for CloudFormation deployment

3. NO MIGRATION FILES IN SCRIPT
   - Script does not include generated Drizzle migration files
   - User runs: pnpm --filter @aissisted/db db:generate
   - This creates packages/db/drizzle/ with SQL migration files

4. ENCRYPTION KEYS ARE OPTIONAL FOR LOCAL DEV
   - Script uses fallback keys for development
   - Production requires proper KEY_ENCRYPTION_KEY setup
   - See .env.example for key generation instructions

5. POSTGRES DATABASE IS REQUIRED
   - Script assumes docker-compose.yml exists in repo root
   - Uses existing "postgres" service definition
   - Or user can use infra/docker-compose.prod.yml
